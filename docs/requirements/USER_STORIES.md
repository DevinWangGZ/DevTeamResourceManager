## 开发人员用户故事

### US-001：作为开发人员，我希望能够登录系统，以便访问我的工作台
- **优先级**：P0
- **验收标准**：
  - 可以使用邮箱/用户名和密码登录
  - 登录成功后跳转到个人工作台
  - 登录失败有明确的错误提示

### US-002：作为开发人员，我希望能够维护我的技能列表，以便展示我的技术能力
- **优先级**：P0
- **验收标准**：
  - 可以添加技能（技能名 + 熟练度）
  - 可以修改已有技能
  - 可以删除技能
  - 修改立即生效，无需审核

### US-003：作为开发人员，我希望能够登记我的业务履历，以便记录我的项目经历
- **优先级**：P0
- **验收标准**：
  - 可以添加项目履历（项目、模块、角色、贡献描述、投入人天）
  - 可以修改已有履历
  - 可以删除履历
  - 投入人天支持小数

### US-004：作为开发人员，我希望能够在任务集市浏览可认领的任务，以便找到适合我的任务
- **优先级**：P1
- **验收标准**：
  - 可以看到所有"已发布"状态的任务
  - 可以按技能、项目等条件筛选
  - 可以搜索任务
  - 列表加载时间 < 2秒

### US-005：作为开发人员，我希望能够认领任务，以便开始工作
- **优先级**：P0
- **验收标准**：
  - 可以认领"已发布"状态的任务
  - 认领后任务状态变为"已认领"
  - 任务分配给我
  - 认领操作有明确反馈
  - 认领时不需要评估，直接认领

### US-005-1：作为开发人员，当任务被派发给我时，我希望能够评估后选择接受或拒绝，以便合理安排工作
- **优先级**：P1
- **验收标准**：
  - 当项目经理派发任务给我时，任务状态为"待评估"
  - 可以查看任务详情（标题、描述、拟投入人天、所需技能等）
  - 可以评估任务后选择"接受"或"拒绝"
  - 接受后任务状态变为"已认领"，任务分配给我
  - 拒绝后任务状态变为"已发布"，可被其他开发人员认领
  - 评估操作有明确反馈
  - 与任务认领（主动认领）不同，派发任务需要评估确认

### US-006：作为开发人员，我希望能够提交已完成的任务，以便记录我的工作量
- **优先级**：P0
- **验收标准**：
  - 可以提交我认领的任务
  - 必须填写实际投入人天
  - 提交后任务状态变为"已提交"
  - 提交后任务不可修改（需特殊权限）

### US-006-1：作为开发人员，我希望任务能够自动按认领人天顺序排期，以便合理安排工作时间
- **优先级**：P1
- **验收标准**：
  - 当我认领或接受任务后，系统自动按认领人天顺序排期
  - 排期计算时自动排除节假日（周末、法定节假日）
  - 排期结果在个人工作台清晰显示
  - 可以看到每个任务的预计开始时间和结束时间
  - 排期基于工作日计算，确保时间安排合理

### US-006-2：作为开发人员，当有紧急任务需要处理时，我希望能够置顶任务，以便优先处理
- **优先级**：P1
- **验收标准**：
  - 可以将当前任务置顶，优先处理
  - 置顶操作简单，一键完成
  - 如果当前有正在处理的B任务，置顶A任务后：
    - A任务排在最前面，优先处理
    - B任务按照未完成的人天向后自动延期
    - 延期计算排除节假日
    - 延期后的B任务及其后续任务自动调整排期
  - 可以取消置顶，恢复原排期顺序
  - 置顶状态在工作台清晰标识

### US-007：作为开发人员，我希望能够查看我的个人负荷情况，以便合理安排工作
- **优先级**：P1
- **验收标准**：
  - 可以看到时间轴视图的负荷分布
  - 负荷状态清晰标识（红/黄/绿）
  - 支持按周/月切换视图

### US-008：作为开发人员，我希望能够查看我的绩效数据，以便了解我的工作成果
- **优先级**：P1
- **验收标准**：
  - 可以看到工作量统计（按项目/模块/时间）
  - 可以看到工作量趋势图
  - 数据准确，可追溯

### US-009：作为开发人员，我希望能够管理我的序列等级和单价，以便准确计算产值
- **优先级**：P1
- **验收标准**：
  - 可以设置我的序列等级（如：初级开发、中级开发、高级开发）
  - 可以设置对应等级的单价（如：初级开发1500元/人天、中级开发2000元/人天、高级开发2500元/人天）
  - 可以修改序列等级和单价
  - 修改立即生效，用于产值计算
  - 序列信息在个人档案中清晰显示

## 项目经理用户故事

### US-101：作为项目经理，我希望能够发布任务，以便分配工作
- **优先级**：P0
- **验收标准**：
  - 可以创建任务（标题、描述、拟投入人天、所需技能）
  - 可以保存为草稿
  - 可以发布任务
  - 发布后任务状态变为"已发布"

### US-102：作为项目经理，我希望能够查看待确认的任务，以便及时确认工作量
- **优先级**：P0
- **验收标准**：
  - 可以看到所有"已提交"状态的任务
  - 可以看到任务详情（包括实际投入人天）
  - 列表清晰，易于查找

### US-103：作为项目经理，我希望能够确认任务完成，以便将工作量汇入统计
- **优先级**：P0
- **验收标准**：
  - 可以确认"已提交"状态的任务
  - 确认后任务状态变为"已确认"
  - 确认后实际投入人天自动汇入开发者工作量统计
  - 确认操作简单，一键完成

### US-104：作为项目经理，我希望能够查看团队负荷情况，以便合理分配资源
- **优先级**：P1
- **验收标准**：
  - 可以看到团队负荷看板
  - 可以看到每个成员的负荷状态
  - 过载人员有预警提醒
  - 支持按人员/项目筛选

### US-105：作为项目经理，我希望能够查看团队关键指标，以便了解团队状况
- **优先级**：P1
- **验收标准**：
  - 可以看到团队总工作量
  - 可以看到任务完成率
  - 可以看到平均投入人天
  - 数据可视化清晰

### US-106：作为项目经理，我希望能够查看待办提醒，以便及时处理重要事项
- **优先级**：P1
- **验收标准**：
  - 可以看到待确认任务数量
  - 可以看到过载人员提醒
  - 可以看到即将到期任务
  - 支持一键处理

### US-107：作为项目经理，我希望能够查看人员信息，以便了解团队成员能力
- **优先级**：P1
- **验收标准**：
  - 可以看到团队成员列表
  - 可以看到每个人的技能和履历
  - 可以按技能、项目等条件筛选
  - 信息清晰易读

### US-108：作为项目经理，我希望能够管理项目的预计产值和任务产值，以便控制项目成本
- **优先级**：P1
- **验收标准**：
  - 可以在项目立项时填写预计产值
  - 可以查看任务产值（包含已完成和未完成的任务）
  - 任务产值 = Σ(任务的实际投入人天 × 开发人员的序列单价)
  - 可以查看需要分配的产值（已完成任务的产值）
  - 需要分配的产值 = Σ(已确认任务的实际投入人天 × 开发人员的序列单价)
  - 当任务产值超出预计产值时，系统自动提醒项目经理
  - 提醒信息包括：当前任务产值、预计产值、超出金额
  - 提醒建议：找业主补充合同或找部门申请产值补贴
  - 产值数据可视化清晰，支持按项目/时间维度查看

### US-109：作为项目经理，我希望能够查看项目任务的执行视图，以便总览全局
- **优先级**：P1
- **验收标准**：
  - 可以看到当前项目所有任务的执行状态
  - 可以按任务状态筛选（草稿、已发布、已认领、进行中、已提交、已确认、已归档）
  - 可以看到每个任务的详细信息（标题、描述、认领人、投入人天、状态等）
  - 可以看到任务的时间线视图
  - 可以看到任务的负荷分布
  - 支持按项目、模块、人员等维度筛选
  - 视图清晰，便于总览全局

## 系统管理员用户故事

### US-201：作为系统管理员，我希望能够管理用户，以便维护系统用户信息
- **优先级**：P0
- **验收标准**：
  - 可以创建用户
  - 可以修改用户信息
  - 可以删除用户
  - 可以设置用户角色

### US-202：作为系统管理员，我希望能够配置基础数据，以便维护系统基础信息
- **优先级**：P0
- **验收标准**：
  - 可以配置项目信息
  - 可以配置模块信息
  - 可以配置其他基础数据

### US-203：作为系统管理员，我希望能够查看系统日志，以便排查问题
- **优先级**：P1
- **验收标准**：
  - 可以查看操作日志
  - 可以查看错误日志
  - 支持按时间、用户等条件筛选
  - 日志信息完整
