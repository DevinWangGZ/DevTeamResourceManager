## 系统简介

DevTeam Manager 是一个面向开发团队的资源管理与协作平台，核心功能包括：

- **任务全生命周期管理**：从创建、发布、认领，到提交、确认、归档
- **工作量透明统计**：基于实际投入人天，数据客观可追溯
- **负荷可视化**：实时掌握个人和团队负荷，避免过载
- **知识沉淀**：Markdown 文章支持团队知识积累与共享

## 登录与角色

### 登录

访问系统首页，输入用户名和密码登录。登录后根据角色自动跳转：
- **开发人员** → 个人工作台
- **项目经理 / 开发组长 / 系统管理员** → 首页

### 角色说明

| 角色 | 主要职责 |
|------|---------|
| 开发人员 | 维护档案、认领任务、记录工作 |
| 项目经理 | 发布任务、确认工作量、管理项目 |
| 开发组长 | 查看团队全局数据、管理人员 |
| 系统管理员 | 用户管理、系统配置 |

## 开发人员使用指南

### 个人档案维护

**路径**：顶部导航 → 个人中心

#### 技能管理
1. 点击"添加技能"
2. 填写技能名称，选择熟练度（熟悉 / 熟练 / 精通）
3. 点击确认，立即生效，无需审核

#### 业务履历管理
1. 点击"添加履历"
2. 填写项目名、模块、角色、贡献描述和投入人天
3. 历史项目经历可手动登记，也可由任务完成后自动同步

#### 序列管理
1. 在个人档案中找到"序列管理"
2. 设置序列等级（如：中级开发）和对应单价（元/人天）
3. 序列单价用于项目产值计算，请根据实际情况填写

---

### 任务集市

**路径**：左侧导航 → 任务集市

#### 浏览与认领任务
1. 在任务集市查看所有"已发布"状态的任务
2. 可按技能、项目、关键词筛选
3. 系统会根据你的技能标签推荐匹配度高的任务
4. 找到合适的任务后点击"认领"，确认后任务分配给你

#### 被派发的任务
- 项目经理可以直接将任务派发给你，任务状态变为"待评估"
- 收到派发任务后，在消息中心查看通知，进入任务详情
- 点击"接受"接下任务，或点击"拒绝"退回（任务重新回到已发布状态）

---

### 任务处理流程

```
认领任务 → 开始工作 → 提交任务（填写实际人天）→ 等待PM确认
```

#### 第一步：认领或接受任务
- 主动认领：在任务集市找到任务，点击"认领"
- 被动接受：PM派发后，在任务详情点击"接受"

#### 第二步：提交任务
1. 在任务管理页找到"进行中"或"已认领"的任务
2. 点击"提交"按钮
3. 填写**实际投入人天**（请如实填写，此数据用于工作量统计和产值计算）
4. 点击确认提交，任务状态变为"已提交"

#### 第三步：等待确认
- PM 确认后，任务变为"已确认"
- 本次任务的实际投入人天自动汇入你的工作量统计

---

### 个人工作台

**路径**：左侧导航 → 工作台

工作台显示：
- **任务汇总**：待处理任务数量，包括待评估、进行中等
- **负荷时间轴**：按周/月展示你的工作负荷分布，颜色含义：
  - 🟢 绿色：空闲（负荷 < 80%）
  - 🟡 黄色：适中（负荷 80%~120%）
  - 🔴 红色：过载（负荷 > 120%）
- **工作量趋势**：近期完成的工作量走势图
- **待办提醒**：需要你处理的待办事项

---

### 任务置顶

当有紧急任务需要优先处理时：
1. 进入任务详情页
2. 点击"置顶"按钮
3. 该任务会排到你工作队列的最前面
4. 其他任务的排期自动向后顺延（自动排除节假日）
5. 不再紧急时，可取消置顶恢复原排期

---

### 工作量统计

**路径**：左侧导航 → 工作量统计

可查看：
- 按时间维度（日/周/月）的工作量趋势
- 按项目/模块分类的工作量分布
- 数据支持导出为 Excel

## 项目经理使用指南

### 项目管理

**路径**：左侧导航 → 项目列表

#### 创建项目
1. 点击"新建项目"
2. 填写项目名称、描述
3. 填写**预计产值**（用于后续产值超支预警）
4. 点击创建

---

### 任务发布与管理

**路径**：左侧导航 → 任务管理

#### 创建任务
1. 点击"创建任务"
2. 填写任务标题、描述（支持 Markdown 格式）
3. 选择所属项目
4. 填写**拟投入人天**（估算工作量）
5. 填写所需技能（逗号分隔，用于任务集市匹配）
6. 设置截止时间
7. 点击"创建任务"保存为草稿，或直接发布

#### 发布任务
- 草稿状态的任务，在操作列点击**"发布"**
- 发布后任务出现在任务集市，开发人员可见并可认领

#### 编辑任务
- 草稿状态的任务，在操作列点击**"编辑"**可修改所有字段
- 已发布的任务不可直接编辑，需先退回草稿

#### 退回草稿
- 对于已发布但需要修改的任务，点击**"退回草稿"**
- 退回后任务从任务集市下架
- 修改完成后重新发布

#### 派发任务
- 在任务详情页点击"派发"，选择指定开发人员
- 被派发人员会收到消息通知，需要评估后接受或拒绝
- 与开发人员主动认领不同，派发任务需要对方确认

---

### 任务确认

当开发人员提交任务后：
1. 在任务管理页或项目经理仪表盘查看"待确认任务"
2. 点开任务详情，核查实际投入人天
3. 点击**"确认"**按钮完成确认（非审批，直接生效）
4. 确认后开发人员的工作量统计自动更新

---

### 项目经理仪表盘

**路径**：左侧导航 → 项目经理仪表盘

仪表盘展示：
- **项目概览**：各项目任务完成情况
- **产值统计**：实际产值 vs 预计产值，超支时自动预警
- **待确认任务**：需要你处理的待确认任务列表
- **团队负荷预览**：团队成员整体负荷分布

---

### 项目任务执行视图

**路径**：项目详情 → 任务执行视图

可查看：
- 当前项目所有任务按状态分类展示
- 任务时间线（预计开始/结束时间）
- 负荷分布（各成员承载情况）

---

### 项目进展数据

**路径**：项目详情 → 项目进展

可查看：
- 任务完成率图表
- 投入人天统计
- 产值明细（按任务、按人员）

## 通用功能

### 消息中心

**路径**：顶部导航栏 → 铃铛图标

消息分为：
- **通知**：任务状态变更（如任务被派发给你、你的任务被确认等）
- **提醒**：待处理事项提醒
- **系统消息**：系统公告

点击消息可直接跳转到对应任务或页面。

---

### 知识分享

**路径**：左侧导航 → 知识分享

#### 阅读文章
- 浏览已发布的团队文章
- 按分类、标签、作者筛选
- 支持全文搜索

#### 写文章
1. 点击"写文章"
2. 使用 **Markdown 编辑器**撰写（实时预览）
3. 支持粘贴/拖拽上传图片（自动转为 Markdown 图片链接）
4. 支持上传 Word、PPT、PDF、Excel 附件
5. 设置分类和标签
6. 点击"保存草稿"随时保存，或点击"发布"对全团队公开

---

### 数据导出

在工作量统计、任务管理等页面，点击**"导出 Excel"**按钮可下载当前筛选结果为 Excel 文件，支持：
- 工作量数据导出
- 任务列表导出
- 绩效数据导出

## 常见问题

**Q：任务认领后怎么开始工作？**
A：认领后任务状态为"已认领"，你可以直接开始工作。如需标记为"进行中"，在任务详情页点击"开始"按钮（可选操作）。

**Q：我填错了实际投入人天，提交后能修改吗？**
A：提交后需要联系项目经理，由 PM 重置任务状态后重新提交。

**Q：任务已发布但内容有误，如何修改？**
A：点击"退回草稿"将任务退回草稿状态，修改后重新发布。

**Q：负荷图显示我是过载状态（红色），怎么办？**
A：过载是基于已认领任务的"拟投入人天"计算的。可与项目经理沟通，调整任务时间安排，或将不紧急的任务推迟。

**Q：我想记录在本平台以外完成的项目经历，怎么操作？**
A：在个人档案的"业务履历"中手动添加，填写项目名称、模块、角色和投入人天即可。

**Q：文章草稿保存在哪里？**
A：文章草稿保存在服务器，登录后在"知识分享 → 我的文章"中可以查看包括草稿在内的所有文章。

**Q：任务排期是怎么计算的？**
A：系统按认领时间顺序自动排期，基于拟投入人天计算，**自动排除周末和法定节假日**，只计算工作日。
