# 手动测试检查清单

## 快速测试流程

### 第一步：准备测试账号

1. **项目经理账号**
   - 用户名：`pm001` 或创建新账号
   - 角色：项目经理

2. **开发人员账号**
   - 用户名：`dev001` 或创建新账号
   - 角色：开发人员

### 第二步：基础功能测试（5分钟）

#### ✅ 测试1：创建任务
- [ ] 使用项目经理账号登录
- [ ] 进入任务管理页面（http://localhost:5173/tasks）
- [ ] 点击"创建任务"
- [ ] 填写任务信息并创建
- [ ] 验证任务出现在列表中，状态为"草稿"

#### ✅ 测试2：发布任务
- [ ] 点击任务"查看"按钮
- [ ] 点击"发布"按钮
- [ ] 验证任务状态变为"已发布"

#### ✅ 测试3：认领任务
- [ ] 退出登录，使用开发人员账号登录
- [ ] 进入任务管理页面
- [ ] 找到刚发布的任务
- [ ] 点击"认领"按钮
- [ ] 验证任务状态变为"已认领"
- [ ] 验证任务分配给自己

#### ✅ 测试4：查看排期
- [ ] 点击任务"查看"进入详情页
- [ ] 查看"排期信息"卡片
- [ ] 验证排期信息显示（预计开始时间、结束时间、工作日数量）

#### ✅ 测试5：提交任务
- [ ] 在任务详情页点击"提交"按钮
- [ ] 填写实际投入人天（如：6.0）
- [ ] 点击"提交"
- [ ] 验证任务状态变为"已提交"

#### ✅ 测试6：确认任务
- [ ] 退出登录，使用项目经理账号登录
- [ ] 找到已提交的任务
- [ ] 点击"确认"按钮
- [ ] 验证任务状态变为"已确认"

### 第三步：高级功能测试（10分钟）

#### ✅ 测试7：任务置顶
- [ ] 使用开发人员账号登录
- [ ] 认领2-3个任务
- [ ] 查看每个任务的排期
- [ ] 将其中一个任务置顶
- [ ] 验证置顶任务的排期调整到最前面
- [ ] 验证其他任务的排期自动向后延期

#### ✅ 测试8：任务搜索
- [ ] 在任务列表页面
- [ ] 使用状态筛选（选择"已发布"）
- [ ] 使用关键词搜索（输入任务标题关键词）
- [ ] 验证筛选结果正确

#### ✅ 测试9：高级筛选
- [ ] 点击"展开高级筛选"
- [ ] 设置创建时间范围
- [ ] 设置投入人天范围
- [ ] 点击"搜索"
- [ ] 验证筛选结果正确

#### ✅ 测试10：任务详情页
- [ ] 进入任务详情页
- [ ] 验证所有信息正确显示
- [ ] 验证操作按钮根据角色和状态正确显示
- [ ] 测试各个操作按钮功能

### 第四步：API测试（可选，5分钟）

#### ✅ 测试11：API文档
- [ ] 访问 http://localhost:8000/docs
- [ ] 使用Swagger UI测试主要API端点
- [ ] 验证API响应正确

## 常见问题快速检查

### 问题1：无法登录
- [ ] 检查后端服务是否运行（http://localhost:8000/health）
- [ ] 检查数据库连接
- [ ] 检查用户是否存在

### 问题2：任务排期未显示
- [ ] 检查任务是否已被认领
- [ ] 检查任务状态是否为"已认领"或"进行中"
- [ ] 检查后端日志是否有错误

### 问题3：前端页面空白
- [ ] 检查前端服务是否运行（http://localhost:5173）
- [ ] 检查浏览器控制台错误
- [ ] 检查网络请求是否成功

## 测试完成标准

所有基础功能测试（测试1-6）通过 ✅
- 任务创建、发布、认领、排期、提交、确认流程完整

所有高级功能测试（测试7-10）通过 ✅
- 任务置顶、搜索筛选、详情页功能正常
